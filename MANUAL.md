## 1. Installation

### System Requirements
- **Node.js 18 or newer** — Download from [nodejs.org](https://nodejs.org)
- **Git** — Required for GitHub URL support
- **Anthropic API Key** — Get one at [console.anthropic.com](https://console.anthropic.com)

### Install via npm
```bash
npm install -g codedoc-agent
```

### Verify installation
```bash
codedoc --version
# Should print: 1.0.0
```

## 2. Setup

### Set your Anthropic API key

**macOS/Linux:**
```bash
export ANTHROPIC_API_KEY="sk-ant-api03-..."
```

To persist it, add the line to your shell profile:
```bash
echo 'export ANTHROPIC_API_KEY="sk-ant-api03-..."' >> ~/.bashrc
source ~/.bashrc
```

**Windows (Command Prompt):**
```cmd
set ANTHROPIC_API_KEY=sk-ant-api03-...
```

**Windows (PowerShell):**
```powershell
$env:ANTHROPIC_API_KEY = "sk-ant-api03-..."
```

## 3. Generating Your First Report

### Option A: Analyze a GitHub repository

```bash
codedoc analyze https://github.com/expressjs/express
```

CodeDoc will:
1. Clone the repo to a temporary directory
2. Scan all source files
3. Map dependencies
4. Generate documentation with AI
5. Produce refactoring suggestions
6. Save everything to `./codedoc-output/`

Then open the report:
```bash
open codedoc-output/index.html     # macOS
xdg-open codedoc-output/index.html # Linux
start codedoc-output/index.html    # Windows
```

### Option B: Analyze a local project

```bash
cd /path/to/your/project
codedoc analyze .
```

Or specify the path directly:
```bash
codedoc analyze ./my-project
```

## 4. Understanding the Output

After running, you'll find this structure in your output directory:

```
codedoc-output/
│
├── index.html                  ← START HERE: Main dashboard
├── dependency-map.html         ← Interactive module graph
├── dep-graph.json              ← Raw graph data (JSON)
├── analysis.json               ← Full analysis data (JSON)
├── refactoring-report.json     ← Refactoring data (JSON)
│
└── docs/
    ├── PROJECT_OVERVIEW.md     ← High-level project description
    ├── README_GENERATED.md     ← Auto-generated README
    ├── FILE_DOCUMENTATION.md   ← Per-file documentation
    ├── API_REFERENCE.md        ← API reference (JS/TS)
    ├── DEPENDENCY_MAP.md       ← Text dependency report
    └── REFACTORING_REPORT.md   ← Full refactoring suggestions
```

### index.html — The Dashboard
Your starting point. Contains:
- **Stats grid** — file count, lines of code, dependency count, refactoring issues
- **Language breakdown** — table showing which languages are used
- **Code health grade** — A–F grade based on issue severity
- **Top refactoring suggestions** — the most important ones highlighted
- **Navigation links** — to all sub-reports

### dependency-map.html — The Graph
An interactive D3.js force-directed graph showing which files import which. You can:
- **Drag** nodes to reposition them
- **Zoom** in/out with the scroll wheel
- **Hover** over a node to see file details
- **Pan** the canvas by clicking and dragging the background

### docs/ — The Markdown Files
These are the files you'll typically commit to your repository or wiki. They contain:
- Human-readable documentation generated by AI
- Mermaid diagrams you can paste into GitHub
- Before/after code examples for refactoring

## 5. Customizing Your Analysis

### Choosing what to include

Skip refactoring to run faster (fewer AI calls):
```bash
codedoc analyze . --no-refactor
```

Skip dependency mapping:
```bash
codedoc analyze . --no-deps
```

Docs only (fastest, minimum API cost):
```bash
codedoc docs .
```

### Controlling file scope

Ignore additional patterns:
```bash
codedoc analyze . --ignore "node_modules,dist,*.test.js,**/__mocks__/**"
```

Set max directory depth (useful for huge monorepos):
```bash
codedoc analyze . --depth 5
```

### Setting output location

```bash
codedoc analyze . -o ./project-documentation
```

### Filtering refactoring by severity

Only show medium+ issues (reduces noise):
```bash
codedoc refactor . --severity medium
```

Options: `low` (all issues), `medium`, `high`, `critical`

## 6. Command Reference

### `codedoc analyze <repo>`
Full pipeline: scan → deps → docs → refactor → export.

```bash
codedoc analyze <path-or-url> [options]

Options:
  -o, --output <dir>       Output directory (default: ./codedoc-output)
  -f, --format <format>    Output format: markdown | html | json (default: markdown)
  --no-refactor            Skip refactoring suggestions
  --no-deps                Skip dependency map generation
  --depth <n>              Max directory depth (default: 10)
  --ignore <patterns>      Comma-separated glob patterns to ignore
  --model <model>          Anthropic model (default: claude-sonnet-4-6)
  -c, --config <file>      Config file path (default: .codedocrc.json)
```

### `codedoc docs <repo>`
Documentation only (no refactoring).

```bash
codedoc docs <path-or-url> [options]

Options:
  -o, --output <dir>       Output directory (default: ./codedoc-output)
  -f, --format <format>    Output format (default: markdown)
  --model <model>          Anthropic model
```

### `codedoc refactor <repo>`
Refactoring analysis only.

```bash
codedoc refactor <path-or-url> [options]

Options:
  -o, --output <dir>       Output directory (default: ./codedoc-output)
  --severity <level>       Minimum severity: low | medium | high (default: low)
  --model <model>          Anthropic model
```

### `codedoc deps <repo>`
Dependency mapping only.

```bash
codedoc deps <path-or-url> [options]

Options:
  -o, --output <dir>       Output directory (default: ./codedoc-output)
```

## 7. Config File Reference

Create `.codedocrc.json` in your project root to set project-specific defaults:

```json
{
  "output": "./docs/generated",
  "format": "markdown",
  "model": "claude-sonnet-4-6",
  "depth": 12,
  "ignore": "node_modules,dist,coverage,*.min.js,**/__snapshots__/**",
  "refactor": true,
  "deps": true,
  "severity": "medium"
}
```

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `output` | string | `./codedoc-output` | Where to write output files |
| `format` | string | `markdown` | `markdown`, `html`, or `json` |
| `model` | string | `claude-sonnet-4-6` | Anthropic model ID |
| `depth` | number | `10` | Max directory depth |
| `ignore` | string | `node_modules,...` | Comma-separated glob patterns |
| `refactor` | boolean | `true` | Enable refactoring analysis |
| `deps` | boolean | `true` | Enable dependency mapping |
| `severity` | string | `low` | Minimum issue severity to report |

CLI options always override config file settings.

## 8. Working with the Dependency Map

### Interactive HTML Map
Open `dependency-map.html` for a force-directed graph:
- **Blue nodes** — TypeScript files
- **Yellow nodes** — JavaScript files
- **Blue nodes** — Python files
- **Node size** — proportional to lines of code
- **Arrows** — show import direction (A → B means A imports B)

### Mermaid Diagram
The `docs/DEPENDENCY_MAP.md` includes a Mermaid diagram. To render it:

1. **GitHub/GitLab** — Paste into a Markdown file, it renders automatically
2. **VS Code** — Install the Mermaid Preview extension
3. **Online** — Use [mermaid.live](https://mermaid.live)

### Resolving Circular Dependencies
If the report lists circular dependencies:

```
REFACTORING_REPORT.md:
### [HIGH] Circular Dependency Detected
File: src/auth/session.js
A → B → C → A
```

**Fix pattern:**
```
Before: A imports B, B imports C, C imports A
After:  Extract shared logic to D; A and C both import D
```

## 9. Working with the Refactoring Report

### Code Health Grades

| Grade | Score Range | Meaning |
|-------|-------------|---------|
| A | 0–50 | Excellent — minimal issues |
| B | 51–100 | Good — a few things to address |
| C | 101–200 | Fair — notable technical debt |
| D | 201–400 | Poor — significant refactoring needed |
| F | 400+ | Critical — major quality issues |

### Prioritizing Suggestions

Work top-to-bottom in `REFACTORING_REPORT.md`:
1. Fix all **critical** issues first (circular deps, security gaps)
2. Address **high** issues (large files, missing tests)
3. Tackle **medium** issues during normal sprints
4. Address **low** issues during code review

### Using Before/After Examples

Each suggestion includes code examples:

```markdown
**Before:**
async function loadUser(id) {
  const user = await db.find(id);  // No error handling
  return user;
}

**After:**
async function loadUser(id) {
  try {
    const user = await db.find(id);
    return user;
  } catch (err) {
    logger.error('Failed to load user', { id, err });
    throw new DatabaseError('User load failed', err);
  }
}
```

## 10. CI/CD Integration

### GitHub Actions

Add to `.github/workflows/docs.yml`:

```yaml
name: Generate Documentation

on:
  push:
    branches: [main]

jobs:
  codedoc:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install CodeDoc Agent
        run: npm install -g codedoc-agent

      - name: Generate documentation
        run: codedoc analyze . --no-refactor -o ./docs/generated
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}

      - name: Commit docs
        run: |
          git config --local user.email "action@github.com"
          git config --local user.name "GitHub Action"
          git add docs/generated/
          git commit -m "docs: auto-update generated documentation" || exit 0
          git push
```

### GitLab CI

```yaml
generate-docs:
  image: node:20
  script:
    - npm install -g codedoc-agent
    - codedoc analyze . --no-refactor -o ./docs/generated
  artifacts:
    paths:
      - docs/generated/
  only:
    - main
```

## 11. Troubleshooting

### "ANTHROPIC_API_KEY not set"
```bash
export ANTHROPIC_API_KEY="your-key-here"
```

### "Failed to clone repository"
- Check your internet connection
- Ensure Git is installed: `git --version`
- Verify the URL is public (private repos require auth token in URL)

### "Path does not exist"
Make sure the path exists and you have read permissions:
```bash
ls -la ./my-project
codedoc analyze ./my-project
```

### Analysis is very slow
- Use `--no-refactor` to skip AI analysis of individual files
- Reduce scope with `--depth 5` 
- Add more ignore patterns: `--ignore "tests,fixtures,mocks"`

### Output is empty or missing docs
- Some files may be too small (<20 lines) to generate docs for
- Try running with `DEBUG=1` for verbose output:
  ```bash
  DEBUG=1 codedoc analyze .
  ```

### Rate limiting errors from Anthropic API
- The tool makes sequential API calls; it doesn't parallel-process
- If you hit rate limits, upgrade your API tier or run on smaller batches
